# Utility class quick reference

Generated by `scripts/gen-utilities.mjs` from the token scales in `src/styles/tokens.css`. Edit the generator (not `utilities.css`) and run `pnpm gen:utilities` to update.

For the why/how, see `src/styles/README.md` and the `## CSS rules` section in `CLAUDE.md`.

### Color tonal palettes (`--c-*`)

Six palettes generated from a brand seed (defined in the COLOR section of `src/styles/tokens.css`). Each has 18 tones at: `0, 5, 10, 15, 20, 25, 30, 35, 40, 50, 60, 70, 80, 90, 95, 98, 99, 100`.

| Namespace | Use |
|---|---|
| `--c-primary-*` | brand accent (seed) |
| `--c-secondary-*` | supporting accent |
| `--c-tertiary-*` | contrasting accent |
| `--c-neutral-*` | surfaces, body text |
| `--c-neutral-variant-*` | outlines, dividers, subdued text |
| `--c-error-*` | destructive |

Token-only — no utility classes. Reference from component CSS:

```css
.my-thing { background: var(--c-primary-95); color: var(--c-primary-20); }
```

For Material role tokens (`--primary`, `--surface`, `--on-primary`, …), keep using BeerCSS's dynamic theme. Don't redefine those.

To regenerate from a new seed: `pnpm gen:colors '#NEWHEX'` and paste into the COLOR section of `tokens.css`.

## Responsive variants

Mobile-first, min-width. Prefixes match BeerCSS's `s` / `m` / `l` mental model.

| Prefix | Applies | Underlying media |
|---|---|---|
| (none) | always | — |
| `m:` | tablet+ (≥601px) | `@media (--tablet)` |
| `l:` | desktop+ (≥993px) | `@media (--desktop)` |

Available on every spacing, gap, and typography utility. (Sizing, ratio, round, container, and color utilities ship without responsive variants — add them in `gen-utilities.mjs` if you need them.)

The colon in the class name is a literal `:` character. CSS selectors escape it as `\:`.

```html
<div class="p-3 m:p-5 l:p-7 text-3 m:text-4 l:text-5 gap-2 m:gap-4">…</div>
```

```css
.p-3      { padding: var(--space-3); }
@media (width >= 601px) { .m\:p-5 { padding: var(--space-5); } }
@media (width >= 993px) { .l\:p-7 { padding: var(--space-7); } }
```

## `@custom-media` reference

Defined in `src/styles/media.css`, resolved at build time by LightningCSS. Use anywhere in `.css` files:

```css
@media (--desktop) { .my-thing { padding: var(--space-7); } }
```

| Name | Resolves to |
|---|---|
| `--mobile` | `(max-width: 600px)` |
| `--tablet` | `(min-width: 601px)` |
| `--desktop` | `(min-width: 993px)` |
| `--motion-ok` | `(prefers-reduced-motion: no-preference)` |
| `--motion-reduce` | `(prefers-reduced-motion: reduce)` |
| `--dark` | `(prefers-color-scheme: dark)` |
| `--light` | `(prefers-color-scheme: light)` |
| `--hover` | `(hover: hover)` |
| `--touch` | `(hover: none)` |

## Scaling factor

`--scale` on `:root` is the global density dial. Default is `1`. Override on any wrapper to rescale every rem-based size/space inside it:

```css
.compact { --scale: 0.875; }   /* tighter */
.cozy    { --scale: 1.125; }   /* looser */
```

Things that ARE scaled: `--size-*`, `--space-*` (the rem-based ones).
Things that are NOT scaled: `--size-px-*`, `--size-fluid-*`, `--size-content-*`, `--size-header-*`, `--size-rel-*`, `--bp-*`, `--container-*`, `--text-*`, `--ls-*`, `--lh-*`, all `--c-*` palettes.
